# Web Service Provider Management System

A small-scale web application for managing service providers, user permissions, and operational activities within an organization.  
The system is designed for environments with fewer than 100 users and focuses on administration, user management, and provider data processing.

---

## 📌 Introduction

This project was developed to simulate a real-world service management platform where administrators can manage suppliers and users efficiently.

The application supports:
- Supplier management
- Role-based authorization
- User account management
- Administrative operations
- CRUD functionalities

The project helped improve backend development skills, database management, and system design thinking.

---

## 🚀 Features

### 👤 Authentication & Authorization
- Login / Logout
- Session management
- Role-based access control
- Admin and User permissions

### 🏢 Supplier Management
- Add supplier information
- Edit supplier details
- Delete supplier records
- Search suppliers

### 👨‍💼 User Management
- Manage user accounts
- Assign permissions
- Restrict unauthorized access

### 📊 Dashboard
- Display system overview
- Manage operational activities

- users
- suppliers
- roles
- permissions

Relationships:
- One admin can manage multiple users
- Users can interact with supplier data depending on permissions
